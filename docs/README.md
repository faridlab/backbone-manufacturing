# manufacturing — Handbook

The documentation set for **`backbone-manufacturing`** — the Backbone Framework domain module that
owns manufacturing: the **product definition** (BOMs, operations, workstations — *what* to build)
and the **execution** (work orders, job cards — the *act* of building), emitting the WIP/FG
valuation postings that turn raw materials + labour into finished-goods inventory value.

> This module began life as the Backbone module skeleton and has since been built out into the real
> `manufacturing` domain — eight entities across two schema models, plus a hand-authored write path
> (`ManufacturingWriteService`) that runs the WIP/GL seam. The handbook below documents that domain
> as it stands today. Because `docs/**` is a `user_owned` path, it lives with the module and is
> regenerated by nothing.

Every page below names **one reader** and **one mode** (Diátaxis) at its top. Find your reader,
follow the path.

## Conventions this handbook follows

- **Reader + mode named** at the top of every page.
- **Commands are real.** Every `metaphor …` command was run against `metaphor 0.2.0` while writing. Where a command in the top-level [README](../README.md) is stale, the handbook flags it and gives the working form.
- **Code wins over docs.** When a doc and the schema/code disagree, the schema YAML (the source of truth) wins — the doc is the bug.
